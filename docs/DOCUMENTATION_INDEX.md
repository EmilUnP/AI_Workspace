# Documentation Index

This project now uses a lightweight documentation model.

## Core Files

- `README.md` - project entry point
- `CHANGELOG.md` - release notes from reset baseline
- `docs/FEATURES.md` - what the product currently provides
- `docs/TECHNICAL.md` - architecture and core modules
- `docs/API.md` - API overview and auth basics
- `docs/DEPLOYMENT.md` - minimal deployment and env notes
- `docs/ROADMAP.md` - near-term priorities

## Principles

- Keep docs short and current.
- Document present state, not legacy history.
- Prefer simple bullets over long narratives.
